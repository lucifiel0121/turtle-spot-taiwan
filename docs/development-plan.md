## Community Consensus & Dissenting Views

| 技術決策 | 社群共識 | 反面意見/已知陷阱 | 來源 |
|----------|---------|------------------|------|
| Pages Router（新專案） | 官方長期支援、文件齊全；既有生態成熟 | 主流意見認為新專案應優先 App Router（RSC、官方推薦）；Pages Router 為使用者明確指定，記錄不改 | Next.js docs / GitHub discussions |
| SWR + graphql-request | 輕量組合，單 query 場景廣被推薦 | Apollo/urql 派認為缺 normalized cache；本案僅一條無參數 query，normalized cache 無收益 | SWR docs、SO 討論 |
| Embla carousel | shadcn 採用後成為 React carousel 主流；plugin 生態（Autoplay/ClassNames）活躍 | tween 效果（scale/opacity）需用 scrollProgress 手寫，官方範例程式碼較 imperative；loop 模式下 slide clone 與 index 對映曾有踩雷 issue | Embla GitHub examples/issues |
| IG 限動自製（不用 react-insta-stories） | 需求僅 dots 連動/箭頭/手勢/進場動態，自製 state + swipe 即可；套件社群評價集中在「客製化受限」 | 自製要處理 touch 事件邊界（swipe 閾值、與垂直捲動衝突）；若日後要自動計時播放進度條，套件較省事 | react-insta-stories GitHub issues |
| Tailwind v4 | 新專案預設；CSS-first config 簡潔 | v4 移除 tailwind.config.js 預設、部分 shadcn 舊範例仍是 v3 語法，混用會踩雷；若審查發現 shadcn init 對 v4+Pages Router 有未解 issue，回退 v3 是安全路徑 | Tailwind blog、shadcn GitHub issues |
| 手寫 GraphQL 型別（不用 codegen） | 單一 query 手寫 interface 是社群普遍做法 | schema 演進時型別會漂移；本案為一次性筆試，接受 | GraphQL codegen docs 討論 |
| Vercel Hobby 部署 demo | 面試作品附 live demo 是前端社群強烈建議的加分做法；Vercel 免費層對 Next.js 零設定 | Hobby plan 限非商業用途（面試作品符合）；免費層有冷啟動與頻寬限制（本案靜態為主影響小）；client 端打第三方 API 需對方 CORS 放行，若被擋需改走 Next API route proxy | Vercel docs / fair use policy、社群作品集慣例 |

## Architecture Notes

### AD-1 資料層：SWR + graphql-request（排除 Apollo Client）
- 選擇：`useSWR('activities', fetcher)`，fetcher 用 graphql-request；`getStaticProps` 給 fallbackData（ISR 3600s）。
- 排除替代：Apollo Client — normalized cache、devtools 對「單一無參數 query」是純 overhead（bundle +30KB+）；urql 同理。
- 相容性：SWR 2.x 支援 React 19；Pages Router 的 getStaticProps 與 SWR fallbackData 是官方 pattern。
- 擴展性：若未來加 mutation/分頁，graphql-request 可續用或平移 urql。
- 效能/安全：ISR 降低首屏 API 依賴；endpoint 公開唯讀無憑證。
- 文件影響：README tech stack 節說明選型理由。

### AD-2 Carousel：shadcn Carousel（Embla）＋手寫 tween（排除 Swiper）
- 選擇：Embla loop + Autoplay plugin；scale/opacity 用 scrollProgress 依官方範例手寫。
- 排除替代：Swiper — 功能齊但 bundle 大、CSS 侵入性強，且 shadcn 已內建 Embla 封裝，混兩套 carousel 增加維護面。
- 相容性：shadcn Carousel 原生支援；觸控內建。
- 擴展性：Embla plugin 架構（ClassNames/Fade/WheelGestures）可隨需求疊加，不需換底層。
- 效能/安全：transform-based 動畫走 compositor，無 layout thrash。
- 文件影響：README 註明 Embla plugin 組合。

### AD-3 IG 限動：自製元件（排除 react-insta-stories）
- 選擇：自製 story viewer（index state + dots + swipe + CSS 進場動畫）。
- 排除替代：react-insta-stories — 預設含自動計時進度條與固定版型，客製成本高於自製；本設計 dots 是「頁數指示」非「計時進度」。
- 相容性：無額外依賴。
- 擴展性：若日後要自動播放計時，state machine 可加 timer。
- 效能/安全：post_link 過協定 allowlist；null 欄位條件渲染。
- 文件影響：元件 API 註解 + README。

### AD-4 Tailwind v4 + shadcn（回退路徑 v3）
- 選擇：create-next-app 預設 Tailwind v4 + shadcn Tailwind v4 指南。
- 排除替代：v3 — 僅在 S0.2 遇 shadcn×v4×Pages Router 未解 issue 時回退（回退成本低：S0.1 重跑 config）。
- 相容性：v4 為 create-next-app 現行預設，與 shadcn 官方指南對齊。
- 擴展性：CSS-first config 下 design tokens 以 `@theme` 管理，後續加 token 無需改 build 設定。
- 文件影響：README 記錄實際採用版本。

### AD-6 部署：Vercel Hobby（排除 Netlify / GitHub Pages）
- 選擇：GitHub repo + Vercel 整合自動部署，README 附 demo link。
- 排除替代：GitHub Pages — 純靜態託管，不支援 ISR/getStaticProps revalidate 與 API route（CORS fallback 會用到）；Netlify — 支援 Next.js 但轉譯層（OpenNext/Netlify runtime）非原廠，Pages Router 相容性風險高於零設定的 Vercel。
- 相容性：Next.js 原廠平台，Pages Router + ISR 零設定；本專案無 secrets（endpoint 公開），不需環境變數管理。
- 擴展性：後續加 API route proxy、preview deployment（PR 預覽）皆原生支援。
- 效能/安全：CDN edge 快取靜態資產；Hobby plan 非商業用途限制與面試作品相容；demo 公開後 repo 內容即公開資訊，不得含任何 secrets（S5.4 已覆蓋）。
- 文件影響：README 頂部 demo link + 部署說明一節。

### AD-5 靜態內容策略（content 常數檔）
- 選擇：資料卡、dive site、footer 等 Figma 內文為靜態硬編（API 僅供 activities）；文案集中 `content/` 常數檔，元件收 props。
- 排除替代：(a) headless CMS / JSON 遠端配置 — 筆試一次性專案，引入 CMS 是純 overhead；(b) 文案散落各元件內 — 逐字核對 Figma 文案時要跨檔找，維護差。
- 相容性：常數檔為純 TS module，與任何元件結構相容。
- 擴展性：language 鈕若日後定義為 i18n 切換，`content/` 集中結構可平移至 next-i18next 的 locale 檔，遷移成本低。
- 效能/安全：build-time 常數，無執行期成本；無使用者輸入。
- 文件影響：README 說明 content 檔結構；design-questions 記錄 language 鈕未定義行為。

## Security / Threat Model

觸及面（輕度）：
1. **外部 API 消費**：endpoint 公開唯讀、無憑證，僅 GET 語意 query；無注入面（query 為常數字串，無使用者輸入插值）。
2. **post_link 外部 URL 渲染**：協定 allowlist（http/https）過濾（防 `javascript:` XSS）；`target="_blank"` 一律搭 `rel="noopener noreferrer"`（OWASP Reverse Tabnabbing）。
3. **API 回應渲染**：一律走 React JSX text node（自動 escape）；全案禁 `dangerouslySetInnerHTML`。
4. 無認證、無 DB、無檔案上傳、無 shell、無反序列化 → 對應 OWASP 面不適用。
落地驗證：S4.2 safe-url 測試 + S5.4 /security-review。
